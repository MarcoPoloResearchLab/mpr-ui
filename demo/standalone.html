How this demo works

  • <mpr-login-button> renders the Google Sign-In button via GIS and handles the credential exchange with TAuth.
  • <mpr-user> displays the authenticated profile and provides a dropdown with menu items and logout action.
  • Both components listen for mpr-ui:auth:* events to stay in sync with the session state.
  • YAML config from ./config.yaml applies the tenant, GIS client ID, auth paths, and shared button styling before mpr-ui.js initializes the custom elements.
  • TAuth endpoints (/auth/nonce, /auth/google, /auth/logout) are proxied through the same origin, and /tauth.js keeps the session state synchronized.
  • This pattern is useful for login pages, sidebars, or anywhere you need auth controls outside the header authentication flow.
