TAuth demo

This page connects the mpr-header web component to a local TAuth service running under Docker Compose. Sign in with Google, observe the authenticated profile state, and use the header profile menu to clear your session.

Current session

Google sign-in requests are exchanged via the TAuth service configured in ./config.yaml. The helper script provided by TAuth monitors /me and /auth/refresh to keep a valid browser session without storing tokens in localStorage.

How it works

  1. The frontend is served by gHTTP on port 4443 (HTTPS), which proxies /auth/* and /me to the local TAuth container.
  2. Auth configuration is loaded from ./config.yaml via the mpr-ui-config.js loader, which applies attributes (google-site-id, tauth-tenant-id, etc.) to <mpr-header> and <mpr-user> elements.
  3. The tauth.js helper is served from the same origin at /tauth.js and initialized with the browser origin plus tenantId from the config.
  4. The header component requests /auth/nonce before rendering the Google button and exchanges the returned credential via /auth/google.
  5. The tauth.js helper refreshes sessions by calling /auth/refresh whenever a request returns 401.
  6. Clicking Log out in the menu invokes /auth/logout and resets the profile snapshot below.
