TAuth demo

This page connects the mpr-header web component to a local TAuth service running under Docker Compose. Sign in with Google, observe the authenticated profile state, and use the header profile menu to clear your session.

Login verification

This shipped <mpr-auth-diagnostics> surface listens to the header auth events and mirrors the current profile snapshot so integration work can verify that login wiring is actually alive.

How it works

  1. The frontend is served by gHTTP on port 4443 (HTTPS), which proxies /auth/* and /me to the local TAuth container.
  2. Auth configuration is loaded from ./config-ui.yaml via the mpr-ui-config.js loader, which applies attributes (google-site-id, tauth-tenant-id, etc.) to <mpr-header> and <mpr-user> elements.
  3. The browser only needs the same-origin auth routes: /auth/nonce, /auth/google, /auth/logout, /auth/refresh, and /me.
  4. The header component requests /auth/nonce before rendering the Google button and exchanges the returned credential via /auth/google.
  5. When /me reports that the session needs renewal, mpr-ui retries through /auth/refresh before updating the shell state.
  6. Clicking Log out in the menu invokes /auth/logout and resets the profile snapshot below.
